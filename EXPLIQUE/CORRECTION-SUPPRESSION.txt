🔧 CORRECTION - SUPPRESSION AUTOMATIQUE DES UTILISATEURS
========================================================

❌ **PROBLÈME IDENTIFIÉ :**
Quand un nouvel utilisateur rejoint la discussion, tous les autres utilisateurs sont automatiquement supprimés de la base de données, même s'ils sont encore connectés et actifs.

🔍 **CAUSES TROUVÉES :**

1. **Délai de nettoyage trop court** : 2 minutes au lieu de 5
2. **Nettoyage agressif** lors des nouvelles connexions
3. **Vérifications incohérentes** entre différentes parties du code
4. **Suppression non-ciblée** dans certaines opérations

🛠️ **CORRECTIONS APPLIQUÉES :**

### 1. Délais uniformisés (5 minutes au lieu de 2)
```php
// AVANT : 2 minutes
WHERE last_activity < DATE_SUB(NOW(), INTERVAL 2 MINUTE)

// APRÈS : 5 minutes  
WHERE last_activity < DATE_SUB(NOW(), INTERVAL 5 MINUTE)
```

### 2. Nettoyage sécurisé avec logs
```php
$cleanedCount = $stmt->rowCount();
if ($cleanedCount > 0) {
    error_log("🧹 Sessions inactives nettoyées: $cleanedCount");
}
```

### 3. Vérifications améliorées avec debug
```php
error_log("🔍 Vérification nom '$username': $otherActiveUser autre(s) session(s) active(s)");
```

### 4. Insertion sécurisée
```php
// Utilisation de ON DUPLICATE KEY UPDATE pour éviter les conflits
INSERT INTO active_users (...) VALUES (...) 
ON DUPLICATE KEY UPDATE last_activity = NOW()
```

### 5. Suppression ciblée uniquement
- Suppression SEULEMENT des sessions spécifiques
- Plus de suppression globale par nom d'utilisateur
- Logs détaillés de toutes les suppressions

4. **Test de nettoyage** :
   - Attendre 5+ minutes sans activité
   - Connecter un nouvel utilisateur
